{# Start of prototype write up #}
Date of prototype
20 December 2017
Feature
Award build and Payment screens
Prototype version
V5

Hypothesis

If we add drop down options for award types to the build an award screen,
then it will enable advisers to easily select the correct item,
because all award types will be clearly presented.

If we make the page inputs relevant to each award type selection,
then advisers will only enter the data needed for that item,
because only enetring data which is needed will make efficiencies processing the application.

If we put more information of the award on to payments views,
then payments staff will be able to check that the payments claim is valid,
because having better information to check against will reduce error and save time.

Changes

Make each selection from the award drop down menu present a tailored input screen relevant to the option chosen.

Explore and create input screens relevant to award item chosen from the drop down menu.

Make the information on the payments screens relevant to the award item the customer has accepted an offer on. Add better payee deatils showing the live payee(s)

Research notes

Designs have been tested with payments staff at Harrow and with SME's

Further work needed to improve the information on the live award views, better use of column headers in the table views

More work needed on the payee deatils
