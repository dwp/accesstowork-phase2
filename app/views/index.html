Date of prototype
20 December 2017
Feature
Award build and Payment screens
Prototype version
V5

Hypothesis

If the award build screens options are in a drop down menu, then it will enable advisers to easily select the correct item to make an award, because it will be clear which award will staisfy the application.

If we make the award inputs relevant to each award item selection, then advisers will only enter the data needed for that item, because enetring data which is not needed will make efficiencies processing the application.

If we put more information of the award details on payments views, then payments staff will be able to check the payments claim better, because having better information to check against the payments will reduce error and save time.

Changes

Make each selection from the award drop down menu present a tailored input screen relevant to the option chosen.

Explore and create input screens relevant to award item chosen fromt he drop down menu.

Make the information on the payments screens relevant to the award item the customer has accepted an offer on. Add better payee deatils showing the live payee(s)

Research notes

Designs have been tested with payments staff at Harrow and with SME's

Further work needed to improve the information on the live award views, bteer use of column headers in the table views

More work needed on the payee deatils
